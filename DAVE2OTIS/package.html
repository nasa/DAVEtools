Overview of the DAVE2OTIS utility package

This package provides a tool to convert DAVE-ML files into OTIS4® input (source) files.

Overview

The gov.nasa.daveml.DAVE2OTIS package builds on parsing capabilities provided in the gov.nasa.daveml.DAVE package to provide a utility to convert Dynamic Aerospace Vehicle Exchange Markup Language (DAVE-ML) files into two OTIS4 source files. See {@link http://daveml.nasa.gov} for more information about DAVE-ML.

gov.nasa.daveml.DAVE itself depends on the org.jdom package, a product of the JDOM project, a Java-based XML document parser. See {@link http://www.jdom.org} for more information about JDOM.

DAVE2OTIS parses a DAVE-ML XML file into an instantiation of the {@link gov.nasa.daveml.DAVE.Model Model} class, auto-verifying the model with any embedded checkcase data. It then creates two files: an OTIS4 _.dat table input deck containing data and an associated FORTRAN language _.f source file containing the appropriate build-up equations. Command-line options allow for creation of a Simulink® library file instead of a model, identification of the model as an "enabled" subsystem, and creation of a model file that is compatible with Simulink® 4.0.

The latest version of DAVEtools can be downloaded from {@link http://dscb.larc.nasa.gov/Products/SW/DAVEtools.html}.

Command Syntax



  java DAVE2OTIS [-count] [-debug] [-warnruntime] [-lib] [-enabled] DAVE_document
where
-count
generates statistics such as # points, functions, signals, etc.,
-debug
indicates descriptive messages are to be emitted,
DAVE_document
is the name of the DAVE-ML file to parse. This file normally has a .xml filetype but that is not required.

Theory of Operation (needs to be rewritten for DAVE2POST)

The static DAVE2OTIS.main() routine

main() is called with the name of the DAVE-ML file (typically ending with .xml) to convert. main() calls the {@link gov.nasa.daveml.DAVE2OTIS.DAVE2OTIS#DAVE2OTIS(java.lang.String[]) DAVE2OTIS()} constructor with the argument string array to create a new DAVE2OTIS object.

The {@link gov.nasa.daveml.DAVE2OTIS.DAVE2OTIS#DAVE2OTIS(java.lang.String[])} constructor

The {@link gov.nasa.daveml.DAVE2OTIS.DAVE2OTIS#DAVE2OTIS(java.lang.String[]) DAVE2OTIS()} constructor initially calls the non-parsing version of the utility {@link gov.nasa.daveml.DAVE.DAVE#DAVE() DAVE()} constructor to initialize flags and to create a {@link gov.nasa.daveml.DAVE.Model} placeholder. DAVE2OTIS then parses the provided arguments (by private method parseOptions()) and emits an error message if it can't understand the arguments.

If the -debug flag was specified on the command line, the new model is told to be verbose.

A call is then made to the new DAVE object's {@link gov.nasa.daveml.DAVE.DAVE#parseFile() parseFile()} method to parse and load the model data contained in the input document; see the Theory of Operation section in the {@link gov.nasa.daveml.DAVE} package for information about loading and parsing the specified DAVE-ML description file into a Model.

If included, a comparison of embedded checkcase data is run by a call to the DAVE object's {@link gov.nasa.daveml.DAVE.DAVE#verify() verify()} method. If the DAVE package is significantly out of date without the verify() method, an error is returned.

If requested, statistical info on parsing is written to the system output.

The DAVE2OTIS's {@link gov.nasa.daveml.DAVE2OTIS.DAVE2OTIS#createModel() createModel()} method is then called to create the Simulink® model-building script and data-creation script files.

The {@link gov.nasa.daveml.DAVE2OTIS.DAVE2OTIS#createModel() DAVE2OTIS.createModel()} method

This method converts the previously parsed {@link gov.nasa.daveml.DAVE.Model Model} into equivalent Simulink® block- and line-building M-file commands.

A new {@link gov.nasa.daveml.DAVE2OTIS.POSTDiagram POSTDiagram} object is created by calling its constructor, {@link gov.nasa.daveml.DAVE2OTIS.POSTDiagram#POSTDiagram(gov.nasa.daveml.DAVE.Model) POSTDiagram()}, passing in the Model as a parameter.

The {@link gov.nasa.daveml.DAVE2OTIS.POSTDiagram#POSTDiagram(gov.nasa.daveml.DAVE.Model) POSTDiagram()} constructor

An {@link java.util.ArrayList ArrayList} of blocks slblocklist is created long enough to hold pointers to each Simulink®-converted {@link gov.nasa.daveml.DAVE2OTIS.POSTBlock POSTBlock}s converted from the provided Model's normal {@link gov.nasa.daveml.DAVE.Block Block}s.

A new {@link gov.nasa.daveml.DAVE2OTIS.MDLNameSpace MDLNameSpace} is created to hold a unique namespace for the POSTBlocks. This object is used to convert Block names into Simulink®-compatible names, as well as to ensure each block has a unique name. {@link gov.nasa.daveml.DAVE2OTIS.MDLNameSpace MDLNameSpace} is a class derived from the {@link gov.nasa.daveml.DAVE.DAVE} package's {@link gov.nasa.daveml.DAVE.NameSpace NameSpace} class to deal with Simulink® naming issues.

An {@link java.util.Iterator Iterator} is created from the Model's {@link gov.nasa.daveml.DAVE.BlockArrayList} and the list of blocks is looped through. For each {@link gov.nasa.daveml.DAVE.Block Block} found, a corresponding {@link gov.nasa.daveml.DAVE2OTIS.POSTBlock POSTBlock} is created. The original Block is notified of it's Simulink doppelganger by calling the Block's {@link gov.nasa.daveml.DAVE.Block#setMask( java.lang.Object ) setMask} method. The name of the original Block may be changed after calling it's {@link gov.nasa.daveml.DAVE.Block#setNameSpace( gov.nasa.daveml.DAVE.NameSpace ) setNameSpace} method. The new {@link gov.nasa.daveml.DAVE2OTIS.POSTBlock POSTBlock} is then added to slblocklist.

The slblocklist is then traversed. Each POSTBlock is told to find it's children, by calling it's {@link gov.nasa.daveml.DAVE2OTIS.POSTBlock#findChildren() findChildren} method; this searches downstream signals and fills the object's children {@link java.util.ArrayList ArrayList}, a list of all immediate children blocks. (A nested diagnostic listing is generate at this step if -debug mode has been requested.) Each of the children is then called to search for their downstream children.

During this traverse, any POSTBlock that masks an instance of {@link gov.nasa.daveml.DAVE.BlockInput BlockInput} or {@link gov.nasa.daveml.DAVE.BlockMathConstant BlockMathConstant} Blocks has it's positions set to column 1 in sequential rows. This is done by calling the POSTBlock's {@link gov.nasa.daveml.DAVE2OTIS.POSTBlock#setPosition( int, int, java.lang.String ) setPosition} method. This method calls each of that POSTBlock's children's {@link gov.nasa.daveml.DAVE2OTIS.POSTBlock#setPosition( int, int, java.lang.String ) setPosition} method in turn to set their row to the current row and to subsequent columns. POSTBlocks with multiple offspring keep track of the row depth and put twin children on parallel rows. This method returns the maximum row depth used in placing children, so the diagram expands as required.

The list of POSTBlocks is traversed a second time, this time to set the high-water mark in terms of deepest row and farthest column in the diagram.

New {@link java.util.ArrayList ArrayList}s for rows, columns, and cells are created based on the size of the diagram. These first two {@link java.util.ArrayList ArrayList}s are then populated with {@link gov.nasa.daveml.DAVE2OTIS.POSTRowColumnVector POSTRowColumnVector} objects which manage blocks and their wiring along the two dimensions of the diagram.

Another traverse of the slblocklist is conducted, this time to encapsulate each {@link gov.nasa.daveml.DAVE2OTIS.POSTBlock POSTBlock} in a newly-minted {@link gov.nasa.daveml.DAVE2OTIS.POSTCell POSTCell}, and assign that POSTCell to the proper pair of row & column POSTRowColumnVector.

The {@link gov.nasa.daveml.DAVE2OTIS.POSTDiagram#POSTDiagram(gov.nasa.daveml.DAVE.Model) POSTDiagram} constructor then returns control to the calling routine, DAVE2OTIS.createModel().

Back to DAVE2OTIS.createModel

Any run-time arguments are interpreted to set the new POSTDiagram's internal flags. A new {@link java.io.File} object is created with the stub name of the input file (thus, myfile.xml's stubname is myfile). The modelname is set to this value, and two constructors are called: {@link gov.nasa.daveml.DAVE2OTIS.POSTFileWriter#POSTFileWriter( gov.nasa.daveml.DAVE2OTIS.POSTDiagram, java.lang.String) POSTFileWriter} and {@link gov.nasa.daveml.DAVE2OTIS.MatFileWriter#MatFileWriter( gov.nasa.daveml.DAVE2OTIS.POSTDiagram, java.lang.String) MatFileWriter}. The POSTFileWriter's {@link gov.nasa.daveml.DAVE2OTIS.POSTFileWriter#writePOSTHeader( java.lang.String ) writePOSTHeader} method is called to write boilerplate header information into a new Simulink® model file named myfile.mdl. The MatFileWriter's {@link gov.nasa.daveml.DAVE2OTIS.MatFileWriter#writeDataHeader( java.lang.String ) writeDataHeader} method is called to write comment lines in a new MATLAB® script file called myfile_setup.m. A script to create a new Simulink® model file is then generated by calling the POSTDiagram's {@link gov.nasa.daveml.DAVE2OTIS.POSTDiagram#createModel( gov.nasa.daveml.DAVE2OTIS.POSTFileWriter, gov.nasa.daveml.DAVE2OTIS.MatFileWriter) createModel} method.

The {@link gov.nasa.daveml.DAVE2OTIS.POSTDiagram#createModel(gov.nasa.daveml.DAVE2OTIS.POSTFileWriter, gov.nasa.daveml.DAVE2OTIS.MatFileWriter) POSTDiagram.createModel} method

The POSTDiagram is scanned by column and row to verify the blocks all know their correct row & column number. Each row is then asked to find height of the tallest block by calling the appropriate row's {@link gov.nasa.daveml.DAVE2OTIS.POSTRowColumnVector#getSize()} method. Columns are likewise sized for their widest block. Then each cell in the column is asked to retrieve it's associated POSTBlock, and that block is asked to generate it's associated M-file block creation statements on the POSTFileWriter and then it's associated data statements through the MatFileWriter by calling {@link gov.nasa.daveml.DAVE2OTIS.POSTBlock#createM( gov.nasa.daveml.DAVE2OTIS.POSTFileWriter, int, int ) createModel} and {@link gov.nasa.daveml.DAVE2OTIS.POSTBlock#writeMat( gov.nasa.daveml.DAVE2OTIS.MatFileWriter ) writeMat}, respectively.

Converting {@link gov.nasa.daveml.DAVE.Signal}s into {@link gov.nasa.daveml.DAVE2OTIS.POSTSignal}s comes next: an {@link gov.nasa.daveml.DAVE.SignalArrayList} sigs is obtained from the Model using {@link gov.nasa.daveml.DAVE.Model#getSignals()} method. This list is traversed and each {@link gov.nasa.daveml.DAVE.Signal} is converted into a {@link gov.nasa.daveml.DAVE2OTIS.POSTSignal} by calling the {@link gov.nasa.daveml.DAVE2OTIS.POSTSignal#POSTSignal( gov.nasa.daveml.DAVE.Signal, gov.nasa.daveml.DAVE2OTIS.POSTDiagram) POSTSignal} constructor, which returns newSig. Then, newSig's {@link gov.nasa.daveml.DAVE2OTIS.POSTSignal#createAddLine( gov.nasa.daveml.DAVE2OTIS.POSTFileWriter ) POSTSignal.createAddLine} method is called to write the add_line creation commands to the model-building script.

Back to DAVE2OTIS.createModel

Back in {@link gov.nasa.daveml.DAVE2OTIS.DAVE2OTIS#createModel()}, the {@link gov.nasa.daveml.DAVE2OTIS.POSTFileWriter#writePOSTFooter(String, String)} and {@link gov.nasa.daveml.DAVE2OTIS.MatFileWriter#writeDataFooter()} methods are invoked to complete the model and data loading script files, respectively. These files are then closed, and control returns to DAVE2OTIS.main().

Back to DAVE2OTIS.main

If checkcase data was embedded in the original DAVE-ML document, a verification script is written by a call to the DAVE2OTIS object's {@link gov.nasa.daveml.DAVE2OTIS.DAVE2OTIS#createVerifyScript createVerifyScript} method.

DAVE2OTIS then exits with confirmation that the .mdl and _setup.m files were successfully written.
